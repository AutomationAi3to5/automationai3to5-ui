♿ Accessibility Guidelines

All components comply with WCAG 2.1 AA standards.

Semantic HTML

✓ Use semantic elements: button, nav, main, section, article
✗ Don't use div for buttons or interactive elements

Keyboard Navigation

✓ All interactive elements must be keyboard accessible
✓ Tab order should be logical (left to right, top to bottom)

Color Contrast

✓ Text: minimum 4.5:1 ratio (normal text)
✓ Large text: minimum 3:1 ratio

Focus Indicators

✓ All focusable elements must show visible focus indicator
✗ Don't remove focus outlines

ARIA Attributes

✓ Add aria-label, aria-describedby when needed
✗ Don't over-use ARIA (semantic HTML is preferred)

Form Labels

✓ Every form input must have associated label
✗ Don't use placeholder as label substitute
